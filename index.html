💬 Configuração de Respostas Prontas (WeHelp)

Antes de configurar as automações, precisamos conferir e ajustar as respostas prontas na WeHelp. Acesse Configurações > Respostas prontas. Você pode adicionar novas ou clicar nos 3 pontinhos nas respostas existentes para editar.

Obrigatório: É necessário ter 3 respostas prontas para o público B2C e 3 para o B2B, cobrindo os 3 status do NPS (Promotor, Neutro e Detrator). Elas são usadas nas automações de clientes que atribuíram nota, mas não deixaram comentários.

Dica: As demais respostas prontas podem ser configuradas para os temas mais citados nas avaliações (ex: cobertura de quadra, metodologia). Isso acelera muito os retornos manuais durante a rodada da pesquisa.

Homologando o Manual (Supabase)

Integração Nuvem

Este manual já está com o código pronto para operar totalmente na nuvem. Com a homologação no Supabase, as edições feitas neste painel deixam de ficar presas ao cache do seu navegador (localStorage) e passam a ser sincronizadas globalmente para todos os usuários. Falta só você preencher suas credenciais — veja o passo a passo abaixo.

🗄️ 1. Criar a tabela no Supabase

No painel do seu projeto Supabase, abra o SQL Editor e rode este script uma única vez:

create table manual_config ( id bigint primary key, data jsonb not null, updated_at timestamptz default now() ); alter table manual_config enable row level security; create policy "Permitir leitura publica" on manual_config for select using (true); create policy "Permitir escrita publica" on manual_config for insert with check (true); create policy "Permitir atualizacao publica" on manual_config for update using (true); insert into manual_config (id, data) values (1, '{}'::jsonb);

🔑 2. Colar suas chaves no arquivo

Em Project Settings > API no Supabase, copie a Project URL e a anon public key. Abra o arquivo .html deste manual em um editor de texto, procure por SUPABASE_URL perto do início do código e substitua os dois valores de exemplo pelos seus. Depois é só subir o arquivo (renomeado para index.html) no GitHub e conectar ao Netlify ou GitHub Pages.

🛡️ Sobre segurança: a chave "anon" do Supabase é pública por natureza (já viaja dentro do próprio HTML) e as regras do script acima permitem leitura e escrita para qualquer pessoa que tenha essa chave. Isso é equivalente ao nível de proteção que a senha de admin do manual já oferece hoje. Se quiser um controle mais forte no futuro (só o admin logado grava), dá para evoluir para autenticação do Supabase.

👣 Como fica o dia a dia

Depois de configurado, edite o manual normalmente pelo link publicado e clique em salvar — não precisa mais exportar HTML nem subir arquivo toda vez. A alteração já chega para todos que acessarem o link.

1
Criar o Banco de Dados

No seu projeto do Supabase, criaremos uma tabela chamada manual_config contendo apenas uma coluna para guardar todos os textos e cores (formato JSON).

2
Pegar as Chaves de Acesso

Você precisará copiar a Project URL e a Anon Key nas configurações de API do seu Supabase. Elas são a ponte entre este manual e a nuvem.

3
Liberar a Leitura Pública

Ajustaremos a segurança (Desativar RLS temporariamente ou criar uma regra simples) para garantir que todos que acessarem o link consigam ler o manual, mas só o administrador consiga gravar.

4
Atualizar o Código

SLA · Área NPS

O Reclame Aqui é um canal complementar de manifestações, gerido pela mesma equipe responsável pelo NPS. Embora tratados pela mesma área, os processos são distintos: o NPS é uma pesquisa estruturada de satisfação, enquanto o Reclame Aqui lida com reclamações públicas e demandas específicas, cada um com seus próprios fluxos e prazos.

📌 Classificação

Origem: Franqueadora ou Franquia (unidade específica).
Temas comuns: Implantação, Atendimento / Qualidade, Financeiro, Comunicação / Suporte.

🔄 Fluxo de Tratativa

  • ✅ Monitoramento: mínimo 3x por semana pelo Analista de NPS.
  • ✅ Análise inicial: ler integralmente, validar histórico e, quando pertinente, correlacionar com feedbacks de NPS (unidades com baixo NPS merecem atenção redobrada).
  • ✅ Acionamento: encaminhar à área responsável com resumo, impacto e prazo.
  • ✅ Resposta pública: primeira manifestação em até 48h úteis (evitar respostas genéricas).
  • ✅ Encerramento: confirmar execução da ação e validar a resposta final. Incentivar avaliação do atendimento.

📝 Registro e Consolidação

Todas as reclamações devem ser registradas em base de controle com: Data, Tema, Unidade, Área responsável, Status, SLA e Resultado.

Arquivar separadamente por ANO/MÊS e TEMA para acompanhamento histórico.

🛑 Moderação de Reclamações

Para reclamações em duplicidade, de outra empresa, com conteúdo impróprio ou de natureza trabalhista, o tratamento será feito por meio do fluxo de MODERAÇÃO.

Fluxo de Moderação:

  1. Cadastro: Inserir o NOME do reclamante ou protocolo no sistema.
  2. Seleção do Motivo: Escolher uma das opções: Duplicidade, Outra empresa, Conteúdo impróprio ou Trabalhista.
  3. Justificativa: Descrever detalhadamente o motivo pelo qual a reclamação deve ser moderada.
  4. Encaminhamento: Enviar o pedido para análise pela equipe responsável pela moderação.

Resultado: Sendo válido o pedido, o Reclame Aqui desativará ou anulará a reclamação, removendo-a da visualização pública e dos indicadores.
